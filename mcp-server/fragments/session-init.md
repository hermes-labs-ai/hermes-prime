## Session conventions (recursive/emergent work)

This is the existing session-init scope as a proof-of-concept of the
`scope_class` parameter. Same content as the default fragment today;
divergence happens when scopes earn their own conventions.

### When to call `hermes-ground`

- 3+ consecutive commits without a user-visible ship.
- A new scope class, taxonomy, or named architecture is proposed mid-session.
- 30+ minutes have elapsed since the last external check during emergent work.
- About to coin a noun-phrase label for something with no implementation at a path.
- A fix-attempt failed and you are tempted to ship a second fix without enumerating ≥3 alternative hypotheses.
- The user signals cognitive-bandwidth overflow.
- Rubric-on-rubric depth ≥2.

### The rule most often forgotten mid-session

**No noun-phrase label until a file exists at a path.**
